运维指南
========

ZSpider 以自托管应用方式运行，因此运行时配置、依赖服务、健康检查和启动方式都应当是显式且环境驱动的。

环境模型
--------

使用 ``ZSPIDER_ENV`` 区分环境：

- ``development``：本地源码开发
- ``testing``：CI 或隔离验证环境
- ``production``：自托管生产环境

关键环境变量
------------

+------------------------------+----------------------------------------+
| 变量                         | 作用                                   |
+==============================+========================================+
| ``ZSPIDER_AMQP_URL``         | RabbitMQ 连接地址                      |
+------------------------------+----------------------------------------+
| ``ZSPIDER_MEMCACHED_SERVERS``| Memcached 服务列表                     |
+------------------------------+----------------------------------------+
| ``ZSPIDER_MONGODB_URI``      | 可选的 MongoDB URI 覆盖                |
+------------------------------+----------------------------------------+
| ``ZSPIDER_MONGODB_HOST``     | MongoDB 主机                           |
+------------------------------+----------------------------------------+
| ``ZSPIDER_MONGODB_PORT``     | MongoDB 端口                           |
+------------------------------+----------------------------------------+
| ``ZSPIDER_WEB_HOST``         | Web 监听地址                           |
+------------------------------+----------------------------------------+
| ``ZSPIDER_WEB_PORT``         | Web 监听端口                           |
+------------------------------+----------------------------------------+
| ``ZSPIDER_SECRET_KEY``       | Flask Session 密钥                     |
+------------------------------+----------------------------------------+
| ``ZSPIDER_MANAGE_PORT``      | Dispatcher 管理端口                    |
+------------------------------+----------------------------------------+
| ``ZSPIDER_MANAGE_KEY``       | Dispatcher 管理密钥                    |
+------------------------------+----------------------------------------+

本地依赖服务
------------

本地开发依赖定义在 ``docker-compose.services.yml`` 中：

- MongoDB
- RabbitMQ
- Memcached

启动方式：

.. code-block:: bash

   make services-up

健康检查与 readiness
--------------------

Web 服务暴露了两个端点：

- ``/healthz``：进程存活检查
- ``/readyz``：MongoDB、RabbitMQ 和 Memcached 的就绪检查

示例：

.. code-block:: bash

   curl http://127.0.0.1:5000/healthz
   curl http://127.0.0.1:5000/readyz

管理员初始化
------------

初始管理员账号需要显式创建：

.. code-block:: bash

   make bootstrap-admin ADMIN_USERNAME=admin ADMIN_PASSWORD=change-me

Web 启动方式
------------

Web 进程现在通过 WSGI 服务器运行，而不是 Flask 开发模式 ``app.run``。

推荐单独启动：

.. code-block:: bash

   make run-web

或启动完整开发栈：

.. code-block:: bash

   make dev
