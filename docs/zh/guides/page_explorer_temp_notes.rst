页面探索临时记录
================

本文用于记录当前“页面探索辅助配置”相关实现、验证结论和已知问题，便于切换环境后继续推进。


适用范围
--------

当前记录覆盖以下内容：

- 入口页探索与文章页探索的现状
- 页面获取方式判断逻辑
- 页面预览与高亮反馈
- 索引页多点标注推断 XPath
- 针对 ``https://mil.news.sina.com.cn/`` 的实际验证结论


当前实现
--------

入口页探索
~~~~~~~~~~

当前入口页探索由 ``zspider/services/page_explorer.py`` 驱动，核心能力包括：

- 同时获取直接请求 HTML 和浏览器渲染 HTML
- 比较两种结果的正文相似度与链接重合度
- 给出“直接抓取”或“浏览器渲染”的建议
- 按推荐获取方式生成候选链接区域、页面标题、预览内容

当前页面预览已经与推荐获取方式保持一致，不再出现“建议动态渲染但预览仍然是静态 HTML”的情况。


文章页探索
~~~~~~~~~~

当前文章页探索继续采用单点模式，围绕以下字段给出候选：

- 标题
- 正文
- 时间
- 来源

文章页预览与入口页一样，按推荐获取方式展示，并支持点击节点后直接回填字段 XPath。


获取方式判断
~~~~~~~~~~~~

当前获取方式判断不是单纯依赖静态启发式，而是分为两层：

1. 优先比较直接请求 HTML 与浏览器渲染 HTML
2. 如果浏览器结果不可用，再回退到脚本数、壳节点、正文长度、链接数量等静态启发式

当前判断结果会直接影响：

- 推荐的抓取程序
- 页面预览来源
- 候选链接区域
- 字段候选来源


页面预览与反馈
~~~~~~~~~~~~~~

当前预览支持两类反馈：

- 初始候选高亮
- 应用候选后按当前 XPath 重新高亮命中节点

高亮现已统一为一套样式，不再区分“默认高亮”和“应用后高亮”两种颜色。


索引页多点标注
~~~~~~~~~~~~~~

当前索引页新增了多点标注模式，交互方式如下：

1. 在预览中点击真实文章链接
2. 将当前链接加入标注列表
3. 至少保留两个标注链接
4. 触发“根据标注推断 XPath”
5. 查看推断结果并决定是否应用

当前多点标注推断接口为：

- ``POST /task/explore/index/infer``

对应服务函数为：

- ``infer_index_xpath()``

当前推断逻辑只在“能覆盖全部已标注链接”的候选中做筛选，并优先返回更聚焦标题层级链接的 XPath。


实际验证结论
------------

验证站点
~~~~~~~~

当前重点验证过的真实站点：

- ``https://mil.news.sina.com.cn/``


站点结构特征
~~~~~~~~~~~~

该站点暴露出当前探索能力的典型难点：

- 索引页单个卡片中同时存在正文链接、评论链接、标签链接
- 列表区域中混有排行、推荐、广告等结构相似块
- 详情页第一个 ``h1`` 不是正文标题，而是站点 Logo
- 正文页存在多个标题类节点，例如 ``channel-logo``、``main-title``、``second-title``


已做的定向收敛
~~~~~~~~~~~~~~

针对新浪军事，当前已经做过两类收敛：

- 索引页候选 XPath 优先收紧到标题层级链接，而不是整块区域中的全部链接
- 文章页标题候选不再机械取第一个 ``h1``，而是优先命中更像主标题的节点

这些收敛对当前站点有帮助，但仍然不能保证对复杂新闻站点稳定覆盖。


当前判断
--------

结合真实站点验证结果，当前结论如下：

- 纯规则方式可以做基础候选召回
- 纯规则方式很难稳定覆盖复杂新闻站点的真实目标区域
- 多点标注比单纯自动推断更可靠，但效果仍受候选空间质量限制
- 仅靠 XPath 规则很难从根本上解决“结构相似但语义不同”的区分问题

当前更合适的定位是：

- 规则层负责召回候选
- 用户交互负责快速确认
- 更复杂的语义判断后续考虑引入大模型辅助


当前建议
--------

如果后续继续推进，建议按下面顺序接手：

1. 不再继续大幅扩张纯规则分支数量
2. 保留当前候选召回与人工确认交互
3. 将大模型引入到“候选排序与选择”层，而不是直接替代抓取与解析
4. 让模型在候选列表、局部 HTML、链接文本、局部结构的基础上判断：

   - 哪个候选列表更像真实新闻列表
   - 哪些链接是正文链接，哪些是评论或标签链接
   - 哪个标题节点是真正文标题
   - 哪个正文容器最像文章主内容

5. 模型输出仍然应落回确定性的候选 XPath，而不是直接生成不可复核的结果
